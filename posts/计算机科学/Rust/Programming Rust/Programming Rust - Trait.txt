Trait 必须导入才能使用

在 Rust 中有两种使用多态的方式：Trait Object 和泛型。

## Trait Object

Rust 中 `dyn +Trait名称` 表示接受所有实现了这个 Trait 的类型。

由于这样的类型很多，`dyn +Trait名称` 的类型大小不固定，Rust 不允许直接将其作为类型。一般要用引用：`&dyn +Trait名称`，这就被称为 Trait Object。

一个实现了特定 Trait 的类型可以转为 Trait Object，但是反过来不可以。

可以组合：`&dyn A+B+C`

Rust 会将实现某个 Trait 的类型自动转换成对应的 Trait Object，比如 `File` 实现了 `Write` 这个 Trait，那么 `&mut File` 会自动转换为 `&mut dyn Write`

**内存布局**

和泛型不同，Rust 在编译期不知道 Trait Object 对应的是什么类型，所以为了能在运行时正确地调用 Trait 上的方法，Rust 需要在 Trait Object 上面记录额外的信息。

![](https://api.file.caiwen.work/picture/2026/04/14/qq_pic_merged_1776140463187.jpg)

Trait Object 本质上是一个由两个指针组成胖指针，一个指向数据，一个指向一个 vtable，vtable 里存放着 Trait 对应的方法的指针。

vtable 是在编译期生成的。对于 `dyn T`，编译器会为所有实现了 `T` 这个 Trait 的类型生成一个 vtable。

**智能指针**

可以使用智能指针包裹 `dyn`，比如 `Box<dyn T>`，此时的 `Box` 的内存布局和普通的 `Box` 不同，在栈上有两个指针，一个指针指向在堆上的数据，一个指针指向 vtable。

反正区别只在于第一个指针指向的是栈上的数据还是堆上的数据。
