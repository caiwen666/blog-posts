1. Ideal-Cache Model

1.1 Overview

后续我们对缓存高效算法的分析就基于这样一个 Ideal-Cache Model 上：只有一个 cache set，缓存的 大小为 MMM，cache line 存储的块大小为 BBB，cache line 的数量总共是 MB\frac{M}{B}BM​。只有这一层缓存。并且我们假设如果出现了 capacity miss 的时候，我们永远能做出最优的替换选择（也就是假设我们知道后续的内存访问情况）。

注意，由于我们这里只有一个 cache set，所以 ideal-cache model 并没有考虑 conflict miss 的情况，这和实际还是有差别的。

image-20250928091614831

然后我们有 LRU Lemma：假设一个程序在大小为 MMM 的 ideal-cache 上发生了 QQQ 次的缓存不命中，那么在一个大小为 2M2M2M 全相连缓存（即只有一个 cache set）的缓存上，使用 LRU 的缓存替换策略，最多只会发生 2Q2Q2Q 次的缓存不命中。这也就意味着在全相连缓存上，我们可以近似认为 LRU 就是最优策略。

Tall-cache assumption：我们一律假设我们的缓存是比较高的，也就是 MB>B\frac{M}{B}>BBM​>B，也就是 M>B2M>B^2M>B2。这不仅符合现实情况，而且理论上也应该是这样。感性地理解为我们缓存肯定是少量多次地去覆盖工作集。考虑说如果我们有一个 short cache，那么可能一个 n2<Mn^2<Mn2<M 的 submatrix 都无法放进缓存中。

1.2 并行

下面的分析基于使用 work stealing 的调度策略的。

我们令 QPQ_PQP​ 为在 PPP 个核心上并行执行时总共产生的缓存不命中次数，SpS_pSp​ 表示 work stealing 策略成功 steal 的次数。那么我们有：

QP=Q1+O(SPMB)Q_P = Q_1 + O(\frac{S_PM}{B}) QP​=Q1​+O(BSP​M​)

因为每次 steal 得来的任务最多需要 MB\frac{M}{B}BM​ 次缓存不命中使得该核上的缓存变为任务被 steal 前的缓存状态。

我们之前分析过，SPS_PSP​ 在期望上大概是 O(PT∞)O(PT_{\infin})O(PT∞​) 的。

这意味着，减少并行时的缓存不命中次数，就等同于减少顺序执行时的缓存不命中次数。
